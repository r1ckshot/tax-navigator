## 5. Юридична рамка (рішення: юрист НІ — бінарно)
- Ustawa o doradztwie podatkowym резервує за ліцензованими: індивідуальні поради/опінії + подання декларацій від імені клієнта. Штраф за практику без ліцензії до 50,000 PLN.
- Наш продукт структурно поза резервованою зоною: показує публічні правила + рахує орієнтир + порівнює. Прецедент: inFakt/ifirma/e-pity роками працюють з формулою "kalkulator ma charakter poglądowy, nie stanowi doradztwa podatkowego".
- ПРАВИЛО ФОРМУЛЮВАНЬ (зашити в UI/звіти/промпти): ніколи "вам краще X" → завжди "порівняння ваших варіантів"; кожен висновок з цитатою джерела; дисклеймер на кожному екрані результату.
- Тригер перегляду: якщо колись додаємо подачу/персональну рекомендацію → юрист ДО запуску фічі (OPEN-RISKS).

## 6. RULES MATRIX v1 — архітектура rules-as-data
Структура: правило = {profile, tax_year, rule_id, formula/params, source_url, verified_at}. Профілі додаються поетапно; MVP = профіль P1.

### P1: "IT-спеціаліст, B2B, податковий резидент Польщі" (tax year 2026)
**Резидентство (art. 3 ustawy o PIT + Objaśnienia MF 29.04.2021):** резидент, якщо АБО >183 днів у PL у році (кожна частина дня = день), АБО центр життєвих інтересів у PL (сім'я/житло/джерело доходу). Спецнорма: подружжя-неукраїнці з тимч. захистом — центр інтересів за письмовою заявою. Подвійна резиденція → тай-брейки Конвенції 1993 (стале житло → центр інтересів → звичайне перебування → громадянство).

**Сценарій A — укр ФОП, живучи в PL (резидент):** PL оподатковує світовий дохід; укр єдиний податок (5% 3-гр + ЄСВ + ВЗ) за Конвенцією 1993 НЕ зараховується (метод пропорційного відрахування покриває PIT-подібні податки, не ЄП) → фактично подвійний тягар + інтерпретації KAS: робота з території PL через ФОП = "stała placówka" → дохід оподатковується в PL. Ризик-статус: 🔴 (штрафи, донарахування). Вивід у продукті: "діапазон найгірший + пояснення чому + що зробити".

**Сценарій B — JDG ryczałt 12%** (IT, PKWiU 62.01): 12% від ПРИХОДУ (витрати не віднімаються); ліміт ryczałt 2 млн €. Składka zdrowotna 2026 (база 9,228.64 zł, GUS 22.01.2026): прихід ≤60k → 498.35 zł/міс; 60–300k → 830.58; >300k → 1,495.04; 50% zdrowotnej віднімається від приходу. ZUS społeczne по етапах: ulga na start (6 міс, тільки zdrowotna) → preferencyjny (24 міс, ~456.19 з хворобовою / 420.86 без) → duży ZUS 1,926.76 zł/міс; Mały ZUS Plus якщо торішній прихід <120k.
⚠️ ПРАПОРЕЦЬ były pracodawca: ті самі послуги роботодавцю, з ким був UoP цього/минулого року → втрата права на ryczałt і ulg ZUS. Обов'язкове питання анкети.

**Сценарій C — JDG liniowy 19%:** 19% від ДОХОДУ; zdrowotna 4.9% від доходу (мін. 432.54 zł/міс), ліміт відрахування zdrowotnej 14,100 zł/рік. Вигідний при витратах ≳15–30%. Break-even vs ryczałt 12%: витрати ~<10% приходу → ryczałt виграє (типовий IT без заліза).

**Сценарій D — JDG skala:** 12% до 120k доходу / 32% понад; kwota wolna 30,000 zł; zdrowotna 9% доходу. Рідко оптимальний для IT-B2B, але база порівняння і кейс низького доходу.

**Сценарій E — інкубатор (Bizky/FBA/Useme-тип):** юрособа інкубатора виставляє фактури; ти на umowa o dzieło з KUP 20% (еф. PIT ~13.6%) або 50% при передачі авторських прав (еф. ~6%); БЕЗ ZUS (= без пенсії/лікарняних — окремо страхуватись); абонемент ~300–349 zł/міс або комісія ~5–6%. Ризики: залежність від юрліца, VAT 23% на послуги інкубатора, стеля масштабу.

**Сценарій F — UoP (довідково):** skala + ZUS роботодавця; для порівняння "скільки втрачаєш/виграєш на B2B".

**Приклад калібрування (прихід 15,000 zł/міс, після ulg, повний ZUS):** ryczałt ~10.5–11.2k на руки; liniowy ~10.0–10.8k; інкубатор ~12–13k (без пенсії!); UoP ~10.7k. ВИВІД ЗАВЖДИ ДІАПАЗОНОМ.
